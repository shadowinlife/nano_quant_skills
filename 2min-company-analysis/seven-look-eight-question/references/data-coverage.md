# Current Data Coverage

本文件描述当前仓库 docs 已经明确可查询的数据边界，用于决定某条“七看八问”规则能否直接落地。

## 直接可查的数据域

### 1. 公司静态画像

- `stk_info`：股票名称、行业、市场类型、上市日期、实控人、地区
- `stk_name_history`：曾用名与变更原因
- `stk_st_daily`：ST / 退市风险状态
- `stk_ah_comparison`：A/H 比价与溢价
- `trade_calendar`：交易日历

### 1.5. 行业分类与同业池

- `idx_sw_classify`：申万行业分类字典（L1/L2/L3）
- `idx_sw_member_all`：申万行业成分股全量快照
- `idx_sw_l3_peers`：按当前申万 L3 分类展开的同类股票标准视图

### 2. 财务三表与核心财务指标

- `fin_income`：营收、净利润、营业利润、EBIT、EBITDA、研发费用、扣非前后利润相关字段
- `fin_balance`：总资产、负债、股东权益、货币资金、存货、应收、固定资产、商誉、带息债务等
- `fin_cashflow`：经营/投资/筹资现金流、自由现金流、现金及现金等价物变化
- `fin_indicator`：ROE、ROA、ROIC、毛利率、净利率、周转率、资产负债率、现金流质量、同比/环比、单季度指标

### 3. 业绩事件与前瞻信号

- `fin_express`：业绩快报
- `fin_forecast`：业绩预告类型、利润区间、变动原因

### 4. 股东结构

- `fin_top10_holders`：前十大股东持股比例与变动
- `fin_top10_float_holders`：前十大流通股东持股比例与变动

### 5. 价格、估值、技术面、资金面、筹码

- `stk_factor_pro`：收盘价、复权价、估值指标、均线、MACD、RSI、KDJ、成交额、市值等
- `stk_moneyflow`：大中小单资金净流向
- `stk_moneyflow_ths`：同花顺口径资金流向
- `stk_margin`：融资融券余额与变化
- `stk_cyq_perf`：筹码成本分位、获利盘比率
- `stk_cyq_chips`：逐价位筹码分布
- `stk_auction_open`：集合竞价数据
- `stk_suspend`：停复牌日历

### 6. 指数与相对强弱参照

- `idx_info`：指数发布方、类别、基期、发布日期
- `idx_daily_dc`：指数日线
- `idx_factor_pro`：指数技术指标
- `idx_quote_dc`：指数涨跌家数、领涨股、总市值
- `idx_member_daily`：指数成分股每日快照

## 基于现有数据可以派生的指标

以下内容虽然不是 docs 里的一列，但可以通过现有字段计算得到：

- 连续多年或连续多个季度的营收、净利、ROE、ROA、OCF 趋势
- CAGR、滚动均值、波动率、回撤、同比/环比稳定性
- 现金流质量，例如 `ocf_to_or`、`ocf_to_profit` 的持续性判断
- 杜邦拆解、杠杆质量、周转效率、偿债能力
- 估值与价格联动，例如估值收缩/扩张与价格表现对照
- 筹码集中、获利盘、资金流与价格趋势的交叉验证
- 股东集中度变化、前十大持股比例变化
- 相对指数强弱，前提是先定义对标指数或同业池

## 当前 docs 没有直接覆盖，落规则前必须讨论

- 分产品/分区域/分客户收入结构
- 年报附注、管理层讨论、审计意见、非结构化文本
- 机构一致预期、卖方预测、目标价
- 基金持仓、北向资金、机构调研等更细粒度持仓数据
- 历史分红明细与派息率序列
- 行业景气度、原材料价格、宏观先行指标
- 年报全文驱动的业务/区域/客户结构语义抽取结果

## 容易被误判为“缺数据”但其实可计算的内容

- 自由现金流：`fin_cashflow` 与 `fin_indicator` 已有字段
- 经营质量：可用毛利率、净利率、ROE、ROA、周转率、现金流质量组合判断
- 财务杠杆：`debt_to_assets`、`debt_to_eqt`、`interestdebt`、`netdebt` 已可支撑
- 增长质量：`tr_yoy`、`or_yoy`、`netprofit_yoy`、`dt_netprofit_yoy`、`ocf_yoy` 已可支撑
- 估值合理性：`pe_ttm`、`pb`、`ps_ttm`、`dv_ttm` 可直接查询，历史分位需额外计算窗口

## 对规则设计的影响

1. 优先设计只依赖当前 docs 的规则。
2. 如果规则需要派生指标，必须先写清楚公式、窗口和异常值处理。
3. 如果规则需要缺失数据，先和用户确认替代指标、采集方式或暂缓落地。
